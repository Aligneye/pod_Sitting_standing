# V3 Hardware Reference — Seeed XIAO nRF52840 BLE

## Board Overview

- **MCU:** nRF52840 (Cortex-M4F, 64 MHz)
- **RAM:** 256 KB
- **Flash:** 1 MB
- **USB:** Native USB-C (programming + serial data)
- **Form Factor:** 21 x 17.5 mm (thumb-sized)
- **Power:** 3.3V logic, 5V from USB-C

## PlatformIO Configuration

The XIAO nRF52840 requires Seeed's custom platform — the standard `nordicnrf52` platform does NOT have the correct board definition.

```ini
[env:xiaoble]
platform = https://github.com/Seeed-Studio/platform-seeedboards.git
board = seeed-xiao-afruitnrf52-nrf52840
framework = arduino
lib_archive = no
build_flags = -DUSE_TINYUSB
lib_deps =
    adafruit/Adafruit TinyUSB Library
```

### Critical Notes

| Item | Detail |
|------|--------|
| Platform | Must use Seeed GitHub repo, NOT `nordicnrf52` |
| Board ID | `seeed-xiao-afruitnrf52-nrf52840` (from Seeed platform) |
| USB Serial | Requires `-DUSE_TINYUSB` flag AND `Adafruit TinyUSB Library` dependency |
| lib_archive | Set to `no` (required for TinyUSB linking) |
| Upload | USB-C direct — no debug probe needed |
| Serial Monitor | USB-C same cable, 115200 baud |

### What does NOT work

- `board = adafruit_feather_nrf52840` — compiles but USB Serial won't enumerate correctly on the XIAO
- `platform = nordicnrf52` without Seeed's board JSON — missing pin mappings
- Omitting `-DUSE_TINYUSB` — Serial object exists but no data flows over USB

## Pin Mapping (I2C for LIS3DH)

| Function | Pin Name | nRF52840 GPIO | Wire |
|----------|----------|---------------|------|
| SDA | D4 | P0.04 | Yellow/White |
| SCL | D5 | P0.05 | Blue/Green |
| VCC | 3V3 | — | Red |
| GND | GND | — | Black |

```cpp
Wire.setPins(4, 5);  // D4=SDA, D5=SCL
Wire.begin();
```

## USB Serial Setup (Firmware Side)

```cpp
// TinyUSB-based serial — requires build flag and library
Serial.begin(115200);
// Wait for USB enumeration (up to 4 seconds)
while (!Serial && millis() < 4000) {}
```

The `while (!Serial)` loop is important — without it, early prints are lost because the USB CDC takes ~1-2 seconds to enumerate after reset.

## LIS3DH Accelerometer

- I2C address: 0x18 (default with SDO/SA0 grounded) or 0x19 (SDO/SA0 high)
- Range: ±2G
- ODR: 100 Hz (hardware), firmware decimates to 50 Hz output
- Library: `Adafruit LIS3DH` + `Adafruit Unified Sensor`

## Data Output (USB Serial)

CSV format at 50 Hz:
```
timestamp_ms,acc_x,acc_y,acc_z
1234,0.5432,9.7123,-0.3211
```

## Host-Side Connection

```python
import serial
ser = serial.Serial("COM5", 115200, timeout=1)
line = ser.readline().decode().strip()
```

Auto-detect port by VID (0x239A for Adafruit BSP) or description containing "nRF" / "XIAO".

## Known Quirks

1. **Double-tap reset for bootloader mode:** If the board appears bricked, double-tap the reset button quickly. The LED turns green and a USB drive (`XIAO-SENSE`) appears for UF2 drag-drop flashing.

2. **USB re-enumeration on reset:** After upload or power-cycle, the serial port disappears briefly (~1-2s) while USB re-enumerates. Python scripts should retry connection or wait.

3. **LED_BUILTIN is active-LOW on some revisions:** `digitalWrite(LED_BUILTIN, LOW)` turns it ON.
